# Bytecode API

## Tagline

Modular Go REST API for authentication, RBAC, user administration, optional Redis caching, optional object storage, and background jobs.

## Purpose

Bytecode API provides a Fiber v3 HTTP API backed by PostgreSQL and GORM. The implemented domain is identity and access management: users, roles, permissions, JWT access tokens, opaque refresh tokens, and profile-picture references.

## Features

- Public authentication endpoints for registration, login, token refresh, and logout.
- JWT middleware for protected API groups.
- RBAC middleware based on role permission names stored in PostgreSQL.
- CRUD endpoints for users, roles, and permissions.
- User self-service endpoints for current profile and permissions.
- PostgreSQL migrations managed by Gormigrate.
- Optional Redis cache for list/detail/permission reads.
- Pluggable local or MinIO object storage provider.
- In-process worker pool for immediate asynchronous jobs.
- Separate worker process for scheduled jobs.
- Swagger UI generated by Swaggo.

## Tech Stack

| Area | Implementation |
| --- | --- |
| Language | Go `1.26.1` |
| HTTP framework | `github.com/gofiber/fiber/v3` |
| Database | PostgreSQL |
| ORM | GORM |
| Migrations | `github.com/go-gormigrate/gormigrate/v2` |
| Authentication | JWT signed with HMAC SHA-256 |
| Password hashing | bcrypt cost `12` |
| Cache | Optional Redis |
| Storage | Local filesystem or MinIO |
| Logging | Uber Zap |
| Tests | Go test, Testcontainers PostgreSQL |

## Directory Structure

```text
cmd/
  api/                 HTTP API entrypoint
  worker/              scheduled worker entrypoint
internal/
  core/                infrastructure: cache, config, database, middleware, server, storage, worker
  features/            feature modules: auth, user, role, permission
  model/               GORM models mapped to migration tables
  shared/              response, errors, pagination, validation, Swagger response types
migrations/            ordered Go migration definitions
pkg/
  hash/                bcrypt helpers
  jwt/                 JWT generation and parsing
  logger/              Zap logger setup
test/
  integration/         API integration tests with Testcontainers
  unit/                service, cache, and storage tests
docs/
  docs.go              generated Swagger Go file
  swagger.json         generated OpenAPI JSON
  swagger.yaml         generated OpenAPI YAML
```

## Quick Start

```bash
cp .env.example .env
go mod tidy
docker compose up -d postgres
make migrate-up
make run
```

The API listens on `http://127.0.0.1:8080` by default.

## Environment Setup

Set database and security values in `.env`. Change `JWT_SECRET` before production. For optional dependencies:

```bash
COMPOSE_PROFILES=redis,minio docker compose up -d
```

## Build Commands

```bash
make build
```

This builds `bin/api` and `bin/worker`.

## Development Commands

```bash
make run
make run-worker
make test
make test-unit
make test-integration
```

## Production Build

```bash
docker build -t bytecode-api .
```

The Dockerfile builds only `cmd/api/main.go`; it does not build the worker binary.
